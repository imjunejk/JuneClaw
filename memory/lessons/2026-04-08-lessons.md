# Daily Lessons - 2026-04-08

## Problems
1. VCP 리밋오더 코드에 4건의 이슈 발견 (코드 리뷰 후 수정)
2. VCP 가격 검증 누락으로 잘못된 가격에 주문 가능성
3. 갭업 브레이크아웃이 갭다운과 동일하게 차단되어 유효한 진입 기회 상실
4. drift_from_scan NameError — import 누락으로 런타임 에러
5. 전략 할당 비율 3번 변경 (70/30 → 50/50 → 60/40) — 의사결정 불안정

## Root Causes
1. VCP 리밋오더: 신규 기능 급하게 추가하면서 엣지케이스 미고려
2. 가격 검증: 주문 로직에서 sanity check 누락 (기존 패턴 미적용)
3. 갭 처리: 갭업/갭다운을 구분 없이 일괄 차단하는 단순 로직
4. import 누락: 리팩토링 시 사용처 추적 미흡
5. 할당 비율: 백테스트 결과 해석과 실전 적용 사이 갈등, 확정 기준 부재

## Solutions Applied
1. 코드 리뷰 4건 수정 + VCP 가격 검증 추가
2. 갭업은 +15%까지 허용, 갭다운만 -5% 기준 차단으로 분리
3. drift_from_scan import 정리
4. 최종 육사 전략 (60/40) 채택

## What Should Have Been Done Better
1. **리밋오더 기능은 TDD로 개발했어야** — 주문 로직은 실패 시 실제 손실 발생
2. **갭 처리 로직을 처음부터 방향별로 설계** — 갭업과 갭다운은 의미가 다름
3. **할당 비율은 사전에 기준 정의 후 1회만 변경** — 같은 날 3번 바꾸는 것은 노이즈

## Pattern Detection
- **[HIGH] 신규 트레이딩 로직 급조 → 버그 4건**: 매매 관련 코드는 반드시 리뷰+테스트 후 머지
- **[MEDIUM] 전략 파라미터 빈번 변경**: 데이터 기반 의사결정 프레임워크 필요
- **[LOW] import 누락 반복**: 리팩토링 시 체크리스트 활용

## Preventative Rules
1. **RULE: 매매 주문 로직 변경 시 반드시 단위 테스트 작성** (mock order → assert price/qty/side)
2. **RULE: 전략 할당 비율 변경은 1일 1회 이내, 변경 사유 문서화 필수**
3. **RULE: 갭/가격 필터 로직은 방향(up/down)별 분리 설계**
4. **RULE: 리팩토링 커밋 전 `python -c "import module"` 로 import 검증**
